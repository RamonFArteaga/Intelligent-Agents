Critical Reflection

This section presents a critical evaluation of my learning experience, individual contribution, collaboration and application of intelligent agent concepts.

Introduction

The Intelligent Agents module strengthened my understanding of how autonomous systems perceive information, make decisions, communicate and act within complex environments. The module also demonstrated that effective agent development requires more than technical functionality. It requires transparency, ethical governance, validation, accountability and careful consideration of the consequences of autonomous decision-making.

My learning developed through academic discussions, collaborative planning and the individual implementation of the Inquiro multi-agent system.

Bias in Autonomous Decision-Making

One of the most important lessons was that intelligent agents may reproduce or amplify bias contained in their training data, retrieval sources or decision rules. An agent may appear objective while still producing unequal or misleading outcomes.

This reinforced the importance of validating inputs, comparing multiple sources and retaining human oversight when agent outputs influence important decisions.

Explainable Artificial Intelligence

Explainability became particularly relevant during the development of Inquiro. A system that produces a report without showing how it reached its conclusions is difficult to evaluate or trust.

Logging, source attribution, structured outputs and validation were therefore essential elements of the final implementation. These mechanisms improved traceability and made the agent workflow easier to inspect.

Responsible Deployment

The module changed my understanding of deployment. A technically functional prototype is not automatically ready for responsible organisational use. Deployment requires access controls, audit trails, testing, human authorisation and clear boundaries around agent autonomy.

Conclusion

Overall, the module developed my ability to connect intelligent agent theory with practical system design. The collaborative proposal strengthened the conceptual foundation of the project, while the individual implementation required me to translate that foundation into a functional and testable system.

The experience reinforced the importance of explainability, validation, responsible deployment and human oversight. These lessons are directly applicable to my professional interest in developing reliable artificial intelligence solutions for industrial environments.

Final Project: Inquiro

Inquiro is an LLM-powered multi-agent system designed to support academic research, analysis, validation and structured report generation.

Project Overview

The original project proposal was developed collaboratively by the team. I individually implemented, tested, modified and documented the final system.

The proposal initially described four agents. During implementation, I introduced an additional Validator Agent to improve output quality, structural compliance and reliability.

Five-Agent Architecture

Planner Agent

Defines the research plan and task sequence.

Research Agent

Collects and organises relevant information.

Analysis Agent

Evaluates evidence and identifies key findings.

Validator Agent

Checks quality, structure and consistency.

Writer Agent

Generates the final structured academic report.

Implementation Improvements

  • Added a dedicated Validator Agent.
  • Implemented structured JSON validation.
  • Added logging and performance metrics.
  • Added automated tests.
  • Implemented query validation.
  • Introduced graceful degradation.
  • Created explicit local model routing.
  • Improved traceability and error handling.
